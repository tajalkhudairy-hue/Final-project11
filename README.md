# Final-project11
Ames Housing Data Analysis Report

1 - Introduction In this project, I worked with the Ames Housing dataset, which contains detailed information about house sales, including features such as size, quality, location, and sale price. The dataset is useful for understanding what factors influence house prices. The main goal of this project was to clean the data, create new useful features, and explore patterns that affect house prices. Specifically, I wanted to understand which variables have the strongest relationship with the sale price and how different features interact with each other.
2- Data Cleaning Summary In Phase 1, I started by loading the dataset and exploring its structure. The dataset contained many columns and some issues that needed to be fixed before analysis. First, I checked for missing values using .isnull().sum(). I identified columns with missing data and handled them depending on the situation. For important columns, missing values were handled carefully to avoid losing too much data. I also checked for duplicate rows using .duplicated(), and no significant duplicates were found. Next, I focused on outliers in the target variable (SalePrice). I calculated the 99th percentile and capped extreme values above this threshold. This helped reduce the effect of unusually high prices that could distort the analysis. I wrapped all these steps into a clean_data() function so that the same cleaning process could be reused in later phases. This made the workflow more organized and consistent.
3- Feature Engineering Summary In Phase 2, I created several new features to improve the dataset and make it more useful for analysis. First, I created a price_per_sqft feature by dividing the sale price by the living area. I added +1 to avoid division by zero. This feature helps compare houses more fairly regardless of size. I also created total_bathrooms, where a half bathroom counts as 0.5. This simplifies multiple bathroom columns into one useful feature. Another feature I created was area_x_lot, which multiplies the living area by the lot area. This interaction feature captures the combined effect of house size and land size. To reduce skewness in the data, I applied a log transformation to the sale price (SalePrice_log). Before the transformation, the distribution was highly skewed, but after applying log1p, it became more balanced. I also created house_age using the difference between the year sold and the year built. Then I grouped it into categories using binning (e.g., new, recent, old houses).
4- Exploratory Data Analysis (EDA) In Phase 3, I explored the data visually and statistically to understand patterns. Distribution Analysis I plotted histograms for key numerical variables like SalePrice, Gr Liv Area, and OverallQual. I noticed that the sale price distribution was right-skewed before transformation, which confirmed the need for log transformation. Boxplots I used boxplots to compare sale prices across categories such as Neighborhood and OverallQual. The results showed clear differences in price depending on location and quality. Correlation Analysis I created a correlation matrix and identified the top features related to sale price. Features like overall quality and living area showed strong relationships with price. Groupby Analysis Using groupby, I calculated the average sale price for each neighborhood. Some neighborhoods had much higher average prices than others, showing that location plays a major role in pricing. Scatter Plot Insights Scatter plots showed a positive relationship between living area and sale price. However, the increase in price becomes less steep for very large houses, suggesting diminishing returns. Additionally, I applied standard scaling to some numerical columns such as living area and sale price to normalize their values. Finally, I checked for highly correlated features and removed redundant ones to avoid duplication and improve data quality.
 5- Mathematical Analysis I also applied some basic mathematical concepts: I calculated the mean and standard deviation of the sale price using NumPy. I standardized values manually and compared them with StandardScaler. I estimated a probability: the proportion of high-quality houses (OverallQual ≥ 8) that have a price above the 75th percentile. This probability showed that higher-quality houses are more likely to have higher prices.
 6-Key Findings From the analysis, I found several important insights: Overall quality is one of the strongest factors affecting house prices. Higher quality houses consistently have higher prices. Living area has a strong positive relationship with price. However, the effect becomes weaker for very large houses. Neighborhood has a significant impact on price. Some neighborhoods have much higher average prices than others.
7-Conclusion and Future Work In this project, I successfully cleaned the data, created meaningful features, and explored patterns in house prices. The results show that quality, size, and location are key factors influencing prices.
